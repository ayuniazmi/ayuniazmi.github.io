Project 01: shipping the Architecture Cost & Tradeoff Generator

The problem

Built the first version of the tool that turns a workload description — traffic pattern, latency sensitivity, data shape, team ops capacity — into three candidate AWS architectures, each with a rough monthly cost estimate and honest tradeoffs. This is close to what I already do informally during vendor evaluations at work; the point of building it was to make that reasoning explicit, repeatable, and demoable instead of living only in my head.

$89Serverless /mo
$115Containers /mo
$140VMs /mo

One example run — small team, steady traffic, relational data. Inputs change the ranking.

How it decides

The fit score isn't a black box: every factor is tied to an input and, where it applies, to a Well-Architected Framework pillar (ops appetite → Operational Excellence, traffic pattern → Cost Optimization, latency → Performance Efficiency, data needs → Reliability). Lock-in is tracked separately since it's a real stakeholder concern that Well-Architected itself doesn't score. The full breakdown is documented on the tool page itself.

What's next

Deliberately scoped down for v1: pricing is simplified, hardcoded, illustrative us-east-1 on-demand numbers — not a live pricing API — and the three architecture patterns are fixed rather than arbitrarily composable. Next up: wire in the real AWS Price List API, and turn the recommended architecture into actual Terraform so the tool's output is provably deployable, not just a diagram.
