Cost model assumptions

  • Simplified us-east-1 on-demand list pricing — no Reserved Instances, Savings Plans, or free-tier credits applied.
  • Lambda: 512MB memory per invocation; API Gateway priced as HTTP API (cheaper tier than REST API).
  • DynamoDB: blended on-demand read+write request-unit cost, assuming roughly one read and one write per request.
  • Fargate: 0.5 vCPU / 1GB per task, sized to handle ~50 req/sec per task; EC2: t3.medium instances, sized to ~40 req/sec per instance. A 1.5× headroom buffer is added for spiky traffic. A 2-task/instance floor is enforced for baseline high availability.
  • RDS: db.t3.medium, single instance, 20GB storage — no Multi-AZ, no read replicas.
  • Data transfer: flat 50KB average response size assumption across all three architectures.

These are deliberately simple, stated assumptions rather than a live pricing API — the goal is to make the tradeoff reasoning visible and adjustable, not to produce a bill-accurate quote.
